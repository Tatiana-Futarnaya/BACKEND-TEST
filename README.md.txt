Консольная программа реализующая функционал формирования чека в магазине.

Используемый стек: Java 17, Gradlr 7.5, среда разработки Intellij Idea, базами данных(БД) Postgresql 9.5,контейнер сервлетов Apache Tomcat 9.0.58, HTML, CSS.

Запуск программы через командную строку(cmd).

1. Перед началом запуска командной строки убедитесь, что на вашем компьютере установлен java 17, gradle 7.5, базами данных Postgresql.
Для проверки введите в командную строку:

- Java -version
должно появиться такое
[:\Users\User>java -version
java version "17.0.2" 2022-01-18 LTS
Java(TM) SE Runtime Environment (build 17.0.2+8-LTS-86)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.2+8-LTS-86, mixed mode, 

-gradle -v
должно появиться такое
------------------------------------------------------------
Gradle 7.6
------------------------------------------------------------

- для БД Postgresql 9.5 после устаноки:
1)Зайти в pgAdmin
2)Создать новый сервер: Файл->Добавить сервер
3) Указать:
-Host:localhost
-post:5432
-Имя пользователя: postgres
-пароль:1234
4)Открываете командную строку и вводите:
psql -U postgres -h localhost (провкпякт установлен psql)
Пароль пользователя postgres:1234
(создает базу данных на сервере)
CREATE DATABASE webapps
  WITH OWNER = postgres
       ENCODING = 'UTF8'
       TABLESPACE = pg_default
       LC_COLLATE = 'Belarusian_Belarus.1251'
       LC_CTYPE = 'Belarusian_Belarus.1251'
       CONNECTION LIMIT = -1;
\q (выход)

!!!!Если не установлено, то ВСЁ(java 17, gradle 7.5, базами данных Postgresql)!!! установите.

2. Скачайте исходные файлы приложения и разместите в удобную для Вас папку. 
У меня проект расположен D:\IdeaProjects\ProjectTest.

3. По умолчанию командная строка открыта на диске С. Нам надо перейти, где лежит проект, поэтому вводим в cmd:
1) cd D:\IdeaProjects\ProjectTest
2) D:
В итоге мы окажемся там где надо.

4. Дальше нам надо собрать проект, вводим в cmd:
gradle build
сообщение "BUILD SUCCESSFUL"

5. Запуская проект и вводим данные:
gradle run --args="3-1 2-5 5-1 card-1234"

!!!!Если хотя бы одно из значений не валидно, то чек не распечатается.

6. На экране появится:
Action List
1. Output a list of products and maps from a file
2. Retrieve a list of products and maps from the database
3. Withdraw a check
4. Exit

В задании было предложено сделать интерфейс. За основу был использован архитектурный паттер MVC(упрощенный).
Использовала Servlet. Данные берутся из статической памяти.

Для запуска скачиваем Tomcat (у меня apache-tomcat-9.0.65.zip-> распаковываем) и переходим в cmd:
1) переходим в папку ../bin ( C:\Users\User\apache-tomcat-9.0.65\bin), вводим:
cd C:\Users\User\apache-tomcat-9.0.65\bin
C:

2)Запускаем:
catalina.bat start

-На экране появится еще одно окно с загрузкой, сверните его.
-Откройте браузер например Google Chrome и в адрес http://localhost:8080/ProjectTest/index.jsp
Отобразится заполненная форма с данными "3-1 2-5 5-1 card-1234" 
- Нажмите кнопку check
- Вы перейдете на новую страницу с адресом
http://localhost:8080/ProjectTest/printCheck?itemId1=3&quantity1=1&itemId2=2&quantity2=5&itemId3=5&quantity3=1&itemIdCard=card&quantityCard=1234

3) Остановить сервер. Надо перейти в cmd и ввести:
catalina.bat stop
